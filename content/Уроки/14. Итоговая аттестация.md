> Завершальний блок перевіряє, чи готовий стажер до першого реального ТЗ. Це справжній екзамен із прохідним балом, розбором кожної відповіді і підказкою, які блоки повторити.

### Модуль 36 — Екзамен: перевірка готовності до першого ТЗ

> [!info] Мета
> на реальних задачах (застосовуємо метод до нового товару — бамбукової обробної дошки) перевірити, чи засвоїв стажер головне з курсу і чи готовий писати перше ТЗ. Прохідний бал — **80%** . Варіанти відповідей перемішуються. Відповідайте на всі питання і натисніть «Перевірити себе» — система покаже результат, розбір кожної відповіді і які блоки повторити. Відповіді зберігаються автоматично.

### Довідково — Умови допуску і типові помилки

#### Умови допуску до першого практичного ТЗ

- розуміє структуру лістингу;
- уміє читати бриф;
- розуміє роль головного фото і реальні заборони для мейна;
- уміє знаходити й аналізувати конкурентів;
- будує ланцюжок «факт → вигода → доказ»;
- знає базову логіку CTR і CR;
- уміє знаходити ризики повернень;
- розуміє пріоритетність УТП;
- розуміє різницю Стандарт / Бренд Сторі / A+;
- відрізняє факт від припущення;
- уміє перевіряти тексти;
- пройшов підсумковий тест за прохідним балом.

#### Банк типових помилок

| Типова помилка | Наслідок |
| --- | --- |
| Логотип / промо-текст («Sale», «Prime Day») на головному фото | Порушення правил Amazon, ризик приховування лістингу |
| Невірна кількість у комплекті | Повернення і погані відгуки |
| Неправильні розміри | Повернення «не те, що очікував» |
| Дублювання УТП | Фото витрачені марно |
| Неприродна англійська (калька) | Недовіра, виглядає як підробка |
| Штучно вставлений ключ | Нечитабельний слоган і нуль користі для SEO |
| Неясний опис сцени | Багато правок, дизайнер угадує |
| Копіювання конкурента | Порушення прав, ризик приховування лістингу |
| Надто загальний референс | Дизайнер робить не те |
| Непідтверджені обіцянки | Ризик блокування і повернення |
| Немає мобільної логіки в A+ | На телефоні не читається |
| Перевантажений візуал | Нічого не зчитується за пару секунд |
| Несуттєве УТП як головне | Слабка пропозиція, низька конверсія |
| Приховування важливого обмеження | Повернення і негативний відгук |
| Невідповідність тексту зображенню | Плутанина і недовіра |

> [!note]
> **Підсумок курсу.** Пройшовши всі блоки, стажер уміє провести товар увесь шлях: прочитати бриф → зібрати Карту товару → проаналізувати конкурентів → розставити пріоритет УТП → розподілити смисли по блоках → писати нативні тексти → заповнити шаблон ТЗ → перевірити себе за чек-листом. Після успішної атестації він готовий до першого реального ТЗ.
